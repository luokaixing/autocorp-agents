# 经验教训（Lessons Learned）

> 用途：记录 AutoCorp 每次迭代的复盘

本目录用于沉淀产品迭代、运营动作、技术实践中的经验教训,避免重复踩坑。每条记录建议覆盖:

- **事件背景**:发生了什么、何时何人
- **预期 vs 实际**:原本期待的结果与实际结果
- **根因分析**:为什么会产生偏差(可附 5 Whys)
- **经验教训**:可复用的认知
- **后续行动**:已采取或计划采取的改进措施

文件命名规范:`YYYY-MM-DD-迭代或事件名.md`,例如 `2026-07-15-SEO-MVP-上线两周复盘.md`。

> 复盘文化是 AutoCorp 的核心准则之一:无指责、重改进、必归档。
