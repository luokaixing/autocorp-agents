# 决策日志（Decisions Log）

> 用途：记录 AutoCorp 所有重大决策及理由

本目录用于沉淀公司层面的重大决策(产品方向、技术选型、商业模式、组织架构等),每条记录应包含:

- **决策内容**:一句话概括
- **背景与动机**:为什么需要这个决策
- **考虑的备选方案**:有哪些选项,为何被否
- **最终决定与理由**:选定方案及核心理由
- **决策日期与参与人**
- **后续复盘**:决策的实际效果(待回填)

文件命名规范:`YYYY-MM-DD-简短主题.md`,例如 `2026-06-27-选择 SEO 赛道作为首发.md`。

> 决策一经记录不可删除,仅可通过新决策文件覆盖或修订,以保留完整决策脉络。
